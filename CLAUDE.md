## Architecture: Three-Tier Fallback Pattern

Every data-dependent script uses the same pattern:

```
1. Try live API  →  2. Read cached JSON file  →  3. Use hardcoded verified values
```

This means all scripts run safely without API keys and all figures regenerate from cached data
even if an API is down. Never break this pattern when editing.

## API Keys

All keys live in **GitHub Secrets only** — never in code or committed `.env` files.

| Secret Name | Source | Required? |
|---|---|---|
| `FRED_API_KEY` | fred.stlouisfed.org/docs/api/api_key.html | Yes (inflation cross-check) |
| `BLS_API_KEY` | data.bls.gov/registrationEngine | Recommended (higher rate limits) |
| `HUD_TOKEN` | huduser.gov/portal/dataset/api-registration.html | Yes (FMR data) |
| `FEC_API_KEY` | api.open.fec.gov | Yes (FEC data) |
| `CENSUS_API_KEY` | api.census.gov/data/key_signup.html | Recommended |
| `CONGRESS_API_KEY` | api.congress.gov | Optional |

For local dev: copy `.env.example` → `.env` and fill in your keys. `.env` is gitignored.

## Key Data Facts

- `dmv_macro_baselines.csv`: exactly 9 rows (MD/VA/DC × 2010/2018/2026). Required columns:
  `Jurisdiction, Year, Max_WBA, Taxable_Wage_Base, Avg_Annual_Wage, Weekly_Housing`
- FEC profiles: 7 members, all `data_quality: "VALID"`. If a profile shows INVALID, run
  `fec_integration_v251d.py` to refresh.
- All per_employee_gap values in employer_contribution_gap.json must be > 0. The frozen wage
  base guarantees this by definition.
